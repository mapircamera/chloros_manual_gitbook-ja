# 画像レイヤー

Chloros 画像ビューアーの「画像レイヤー」ドロップダウンメニューを使用すると、元のキャプチャ画像から処理済みの反射率出力、計算された屈折率画像に至るまで、同じ画像の異なるバージョンをすばやく切り替えることができます。

## 画像レイヤーとは？

Chlorosにおいて、**レイヤー**とは、単一のソース画像に対して利用可能なさまざまな画像出力を指します。画像を処理すると、Chlorosは複数のバージョンを作成します：

* **オリジナル画像**（カメラからのJPGおよびRAWファイル）
* **反射率補正済み**出力（反射率補正が有効だった場合）
* **ターゲット画像**（画像にキャリブレーションターゲットが含まれている場合）
* **インデックス画像**（NDVI、NDRE、GNDVIなど、インデックスが設定されていた場合）

イメージビューアーの右上にある**レイヤーセレクターのドロップダウン**を使用すると、ビューアーを離れることなく、これらのバージョンを瞬時に切り替えることができます。***

## 利用可能なレイヤータイプ

### JPG

* カメラからの元のJPGプレビュー画像
* すべての画像で常に利用可能
* カメラで撮影されたままの未処理の状態
* 読み込みと表示が最も高速

**表示するタイミング:**

* 撮影元の画像を素早くプレビューする場合
* 構図やフレーミングを確認する場合
* 処理前に撮影品質を確認する場合

### RAW (オリジナル)

* カメラからの元のRAWセンサーデータ
* デベイヤー処理済みで、後処理は適用されていない
* JPGよりも高いビット深度（通常は12ビットまたは14ビットのセンサーデータ）

**表示するタイミング：**

* 元のセンサーデータの品質を確認する際
* センサーの問題やアーティファクトを確認する際
* 処理前後の結果を比較する際

### RAW (ターゲット)

* キャリブレーションターゲットが含まれていると識別された画像にのみ表示されます
* ターゲットが検出された元のRAW画像を表示します
* ターゲットの検出が成功したことを確認するために使用します

**表示するタイミング：**

* キャリブレーションターゲットが正しく検出されたことを確認する際
* ターゲット画像の品質を確認する場合
* キャリブレーションの問題をトラブルシューティングする場合

{% hint style="info" %}
**ターゲットレイヤー**：このレイヤーは、キャリブレーションターゲットを含む画像のドロップダウンメニューにのみ表示されます。通常の撮影画像にはこのオプションはありません。
{% endhint %}

### RAW (反射率)

* キャリブレーション済みの反射率出力画像
* ヴィネット補正済み（処理で有効化されている場合）
* ターゲットデータを使用した反射率キャリブレーション済み（有効化されている場合）
* すべてのカメラチャンネルを含むマルチバンド画像（TIFF）
* ピクセル値は反射率（パーセント）を表します（パーセントモード使用時）
* [インデックス/LUTサンドボックス](index-lut-sandbox.md) での編集準備完了

**表示するタイミング：**

* キャリブレーション結果の確認
* キャリブレーション品質の検証
* 科学的精度のためのピクセル値の確認
* キャリブレーションの効果を確認するためのオリジナル画像との比較

{% hint style="success" %}
**推奨**：科学的測定や分析のためにピクセル値を確認する際は、RAW（反射率）レイヤーを使用してください。
{% endhint %}

### RAW (NDVI インデックス)... および類似のもの

* 算出された植生インデックス画像（この例では NDVI）
* インデックス名は、処理中にどのインデックスが設定されたかによって変わります
* 例：RAW (NDVI インデックス)、RAW (NDRE インデックス)、RAW (GNDVI インデックス) など
* インデックスの計算結果を示す単一バンドのグレースケール画像
* プロジェクト設定で設定されたインデックスごとに 1 つのレイヤーが表示されます

**使用可能なインデックス名：**

* RAW (NDVI インデックス)
* RAW (NDRE インデックス)
* RAW (GNDVI インデックス)
* RAW (OSAVI インデックス)
* RAW (EVI インデックス)
* RAW (SAVI インデックス)
* その他多数... ([マルチスペクトル・インデックスの計算式](../project-settings/multispectral-index-formulas.md) 参照)

**表示するタイミング:**

* インデックスの計算結果を確認する場合
* インデックス値の範囲を確認する場合
* 関心領域を特定する場合
* GISや分析に使用する前にインデックス画像を検証する場合

## レイヤーセレクターの使用方法

### ドロップダウンを開く

1. 画像を全画面モードで開きます（イメージビューア内の任意のサムネイルをクリックします）
2. ビューアの右上隅にある**レイヤードロップダウン**を探します
3. ドロップダウンには現在選択されているレイヤー（例：「JPG」）が表示されます
4. ドロップダウンをクリックすると、利用可能なすべてのレイヤーが表示されます

### レイヤーの切り替え

1. レイヤードロップダウンをクリックしてリストを開きます
2. 現在の画像で利用可能なすべてのレイヤーが表示されます
3. 任意のレイヤー名をクリックして、そのバージョンに切り替えます
4. 画像は即座に更新され、選択したレイヤーが表示されます

**クイック切り替え：**

* ドロップダウンは前回の選択内容を記憶します
* 次の画像に移動する際、Chlorosは同じレイヤータイプを表示しようとします
* 次の画像にそのレイヤーが存在しない場合、デフォルトでJPGが表示されます

### レイヤーの利用可能性

すべての画像で全てのレイヤーが利用可能とは限りません：

**常に利用可能：*** ✅ JPG （すべての画像にJPGプレビューがあります）

**条件付きで利用可能：**

* ⚠️ RAW（オリジナル） - 画像がRAWまたはRAW+JPGモードで撮影された場合のみ
* ⚠️ RAW（ターゲット） - 画像にキャリブレーションターゲットが検出された場合のみ
* ⚠️ RAW（反射率） - 反射率キャリブレーションを有効にして処理した後のみ
* ⚠️ RAW (\[インデックス] インデックス） - インデックスが設定された状態で処理された後のみ

***

## レイヤーの保持

### 画像間の移動

別の画像に移動する場合（矢印キーを使用するか、サムネイルをクリック）：

**レイヤーの設定は保持されます：**

* 「RAW (反射率)」を表示している場合、次の画像は「RAW (反射率)」 （利用可能な場合）
* 「RAW (NDVI インデックス)」を表示している場合、次の画像は「RAW (NDVI インデックス)」となります（利用可能な場合）
* 同じレイヤーが存在しない場合、デフォルトでJPGが表示されます

**ワークフローの例：**

1. 画像1を開き、RAW (NDVI Index)
2. → キーを押して画像2を表示
3. 画像2には自動的に RAW (NDVI Index) レイヤーが表示されます
4. ナビゲーションを続行すると、すべての画像に NDVI レイヤーが表示されます
5. 多数の画像にわたるインデックス結果の確認に非常に効率的です

***

## 一般的なワークフロー

### ワークフロー 1: 比較（前/後）

**目的**: オリジナル画像と補正済み画像を比較する

1. 処理済み画像を Image Viewer で開く
2. ドロップダウンから **RAW (Original)** を選択
3. ヴィネット効果と未補正の値を確認
4. ドロップダウンから **RAW (Reflectance)** に切り替える
5. 比較 - ヴィネットが除去され、値がキャリブレーション済み

### ワークフロー 2: インデックスの確認

**目的**: データセット全体の NDVI 結果を素早く確認する

1. 最初に処理された画像を開く
2. ドロップダウンから **RAW (NDVI インデックス)** を選択
3. → 矢印キーを使用して次の画像に移動
4. NDVIレイヤーが自動的に保持されます
5. すべての画像を順に確認し、NDVIのパターンをチェックします
6. **RAW (NDRE Index)** に切り替えて比較します

### ワークフロー 3: ターゲットの検証

**目的**: すべてのターゲット画像が正しく検出されていることを確認する

1. ターゲット画像に移動する
2. ドロップダウンから **RAW (Target)** を選択する
3. キャリブレーションターゲットがはっきりと確認でき、検出されていることを確認する
4. 次のターゲット画像に移動する
5. すべてのターゲットについて検証を繰り返す

### ワークフロー 4: ピクセル値の検査

**目的**：科学的な正確性を確保するため、反射率値を確認する

1. 処理済み画像を開く
2. **RAW (Reflectance)** レイヤーを選択する
3. **Pixel Percent** モードを有効にする（右上のツールバーにあるボタン）
4. カーソルを植生エリア上に移動させる
5. ピクセル値が想定範囲内にあることを確認する（NIRの場合は30～70%、Redの場合は5～15%）
6. 土壌および水域の値が適切であるか確認する

***

## レイヤーごとのピクセル値の理解

レイヤーごとに表示されるピクセル値の範囲は異なります：

### JPGレイヤー

* **範囲**: 0～255（8ビット）
* **意味**: 表示値、ガンマ補正済み
* **用途**: 目視確認のみ。科学的測定には使用しない

### RAW（オリジナル）

* **範囲**: 0～65535（16ビット）
* **意味**: センサーの生データ
* **用途**: センサー性能の確認用（キャリブレーション済みではない）

### RAW（反射率）

* **範囲**: 0～65,535（16ビット TIFF）または 0.0～1.0（32ビット パーセント）
* **意味**: 校正済みの反射率（パーセント）
* **用途**: 科学的な測定および分析**16ビット TIFFの場合:**65,535で割ることで反射率（パーセント）が得られます**32ビット パーセントの場合:** 値は直接パーセントを表します（0.5 = 50%の反射率）

### RAW（インデックス画像）

* **範囲**: インデックスによって異なる（通常、正規化インデックスの場合は -1.0 ～ +1.0）
* **意味**: インデックス計算結果
* **例**:
  * NDVI: -1 ～ +1（植生は通常 0.4 ～ 0.9）
  * NDRE: -1 ～ +1（ストレス検出）
  * EVI: 0 ～ 1（植生強調）

## ヒントとベストプラクティス

### レイヤーの効率的な切り替え

* **キーボードショートカットの活用**: レイヤー専用のショートカットはありませんが、ナビゲーションキー（←/→）はすべてのレイヤーで機能します
* **一貫したワークフロー**: 1つのレイヤー（例: NDVI）を選択し、別のレイヤーに切り替える前にデータセット全体を確認してください
* **迅速な比較**: 「オリジナル」と「反射率」を切り替えて、処理品質を確認してください

### パフォーマンスに関する考慮事項

* **JPGが最も高速に読み込まれる**：多数の画像を素早く閲覧する場合に使用
* **RAWレイヤーの読み込みは遅い**：解像度とビット深度が高い
* **インデックスレイヤー**：反射率レイヤーと同等の速度
* **初回読み込みが最も遅い**：同じレイヤーのその後の表示はキャッシュされ、高速になる

### 品質確認

* **常にRAW（オリジナル）を確認**：処理済み出力を信頼する前に、ソースデータの品質を確認してください
* **レイヤーの比較**：レイヤーの切り替え機能を使用して、処理が正しく行われたかを確認してください
* **インデックス範囲の確認**：インデックスレイヤーで「ピクセルパーセント」モードを使用し、値が妥当であるかを確認してください***

## トラブルシューティング

### レイヤーが表示されない

**問題**：期待したレイヤーがドロップダウンに表示されない**考えられる原因：**

* 画像が処理されていない（JPGおよびRAW（オリジナル）のみ利用可能）
* 処理中に反射率キャリブレーションが無効になっていた
* プロジェクト設定で特定のインデックスが設定されていない
* 画像がターゲット専用画像である（ターゲットにはインデックスが生成されない）

**解決策：**

1. 画像が処理されたことを確認する（出力フォルダに処理済みファイルがあるか確認する）
2. プロジェクト設定を確認し、インデックスが設定されていることを確認する
3. 希望するインデックスを有効にして再処理する

### 誤ったレイヤーが表示される

**問題**：予期しないレイヤーで画像が開く**原因**：前の画像のレイヤー設定が引き継がれたが、現在の画像にはそのレイヤーが存在しない**解決策**: 優先レイヤーが利用できない場合、Chlorosは自動的にJPGに切り替わります。これは正常な動作です

### キャリブレーションターゲットが表示されない

**問題**: RAW（ターゲット）レイヤーにターゲット検出が表示されない**考えられる原因:**

* 処理中にターゲットが検出されなかった
* 画像に実際にターゲットが含まれていない
* ターゲット検出の設定が厳しすぎる

**解決策：**

1. デバッグログで「Target found」というメッセージを確認する
2. 画像に実際に目視可能なキャリブレーションターゲットが含まれているか確認する
3. プロジェクト設定でターゲット検出の設定を調整する
4. [ターゲット画像の選択](../processing-images-gui/choosing-target-images.md)を参照する

## 関連機能

### 画像ビューアツール

どのレイヤーを表示している場合でも、以下の機能を使用できます：

* **ズームコントロール**：拡大して詳細を確認
* **パン**：クリックしてドラッグし、ズームした画像内を移動
* **ピクセル値の確認**：カーソルの位置の値を確認
* **ナビゲーション矢印**：レイヤーを維持したまま画像間を移動
* **ピクセルパーセントモード**：DN表示とパーセント表示を切り替え

画像ビューアーの完全なドキュメントについては、[画像の全画面表示](opening-an-image-full-screen.md)を参照してください。

### インデックス/LUT サンドボックス

インタラクティブなインデックステストと可視化を行うには：

* **リアルタイムインデックス計算**：さまざまなインデックス式をテストします
* **LUT カラーマッピング**：グレースケールインデックスにカラーグラデーションを適用します
* **可視化結果のエクスポート**：カラー化されたインデックス画像を保存します

詳細については、[インデックス/LUT サンドボックス](index-lut-sandbox.md) をご覧ください。
