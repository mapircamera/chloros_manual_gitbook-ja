# プロジェクト設定の調整

画像を処理する前に、ワークフローの要件に合わせてプロジェクト設定を構成することが重要です。「プロジェクト設定」 <img src="../.gitbook/assets/icon_project-settings.JPG" alt="" data-size="line"> パネルでは、キャリブレーション、処理オプション、マルチスペクトルインデックス、エクスポート形式などを包括的に制御できます。

## プロジェクト設定へのアクセス

1. Chlorosでプロジェクトを開きます
2. 左サイドバーの**プロジェクト設定** <img src="../.gitbook/assets/icon_project-settings.JPG" alt="" data-size="line"> アイコンをクリックします
3. プロジェクト設定パネルにすべての設定オプションが表示されます

{% hint style="info" %}
**設定はプロジェクトと共に自動的に保存されます**。プロジェクトを再開すると、すべての設定が復元されます。
{% endhint %}

***

## 一般的なワークフローのためのクイックセットアップ

### デフォルト設定（ほとんどのユーザーに推奨）

一般的な MAPIR Survey3 カメラのワークフローでは、デフォルト設定で問題なく動作します：

* ✅ **ヴィネット補正**：有効
* ✅ **反射率キャリブレーション**：有効 （MAPIRターゲットの画像が必要です）
* ✅ **デベイヤー法**: 標準（高速、中品質）
* ✅ **エクスポート形式**: TIFF（16ビット）

画像をインポートし、これらのデフォルト設定で処理を開始するだけです。

***

## プロジェクト設定の概要

「プロジェクト設定」パネルはいくつかのカテゴリに分類されています。各セクションの概要は以下の通りです。詳細なドキュメントについては、[プロジェクト設定](../project-settings/project-settings.md)を参照してください。

### ターゲット検出

Chlorosが画像内のキャリブレーションターゲットをどのように識別するかを制御します。

**主な設定項目:*** **最小キャリブレーションサンプル領域**: ターゲット検出のサイズ閾値（デフォルト: 25ピクセル）
* **最小ターゲットクラスタリング**: ターゲット領域をグループ化する類似性の閾値（デフォルト: 60）**調整が必要な場合:**

* 誤検出が発生する場合はサンプル領域を拡大する
* ターゲットが検出されない場合は、サンプル領域を縮小してください
* ターゲットが複数の検出に分割されている場合は、クラスタリングを調整してください

### 処理

主な画像処理およびキャリブレーションオプション。

**主な設定:*** **ヴィネット補正**: 画像の端でのレンズの暗さを補正します ✅ 推奨
* **反射率キャリブレーション**: キャリブレーションターゲットを使用して値を正規化します ✅ 推奨
* **デベイヤー法**：RAWデータを3チャンネルのマルチスペクトルデータに変換するアルゴリズム
* **再キャリブレーションの最小間隔**：キャリブレーションターゲットを使用する間隔（0 = すべて使用）**詳細設定：*** **光センサーのタイムゾーンオフセット**：PPKの時刻同期用（デフォルト：0）
* **PPK補正を適用**: .daqファイルからのGPS/露出ピンデータを使用
* **露出ピン1/2**: デュアルカメラ構成において、カメラを露出ピンに割り当てます

### デベイヤー方式

現在、Chlorosでは2つのデベイヤー方式を提供しています:

#### 標準 (高速、中程度の品質)

標準デベイヤーは処理が高速ですが、デベイヤー処理によるカラーノイズが発生し、画像の精度が低下し、ノイズが多くなります。

#### テクスチャ対応（低速、最高品質） \[Chloros+ のみ]

テクスチャ対応モードでは、高品質なエッジ認識デベイヤーとAI/MLノイズ除去モデルを組み合わせており、デベイヤー処理によるノイズをほぼ完全に除去します。テクスチャ対応モデルの実行にはGPUメモリ（VRAM）が必要です。処理速度を向上させるため、4GB以上のVRAMが利用可能な場合にこのモードの使用をお勧めします。

### インデックス（マルチスペクトルインデックス）

計算およびエクスポートする植生インデックスを設定します。

**指数の追加方法:**

1.**「指数の追加」**ボタンをクリックします
2. ドロップダウンメニューから指数を選択します（NDVI、NDRE、GNDVI など）
3. 視覚化設定（LUTの色、値の範囲）を設定します
4. 必要に応じて複数の指標を追加します

**よく使われる指標：*** **NDVI**：一般的な植生の健康状態（最も一般的）
* **NDRE**：RedEdgeとの組み合わせによる早期ストレス検出
* **GNDVI**: クロロフィル濃度に敏感
* **OSAVI**: 可視光域の土壌と相性が良い
* **EVI**: 葉面積指数（LAI）が高い領域（LAI）**カスタム式（Chloros+ のみ）：**

* カスタム多波長インデックス式を作成
* すべての画像チャンネルでバンド演算を使用
* 再利用のためにカスタム式を保存

利用可能なすべてのインデックスと式については、[多波長インデックス式](../project-settings/multispectral-index-formulas.md)を参照してください。

### エクスポート

出力ファイルの形式と品質を制御します。

**利用可能な形式：*** **TIFF (16ビット)**：GISおよび科学分析に推奨（0～65,535の範囲）
* **TIFF (32ビット、パーセント)**： 浮動小数点形式の反射率値（0.0～1.0の範囲）
* **PNG (8ビット)**: 視覚化用のロスレス圧縮（0～255の範囲）
* **JPG (8ビット)**: ファイルサイズが最小、ロス有りの圧縮（0～255の範囲）***

## 設定の保存と読み込み

### プロジェクトテンプレートの保存

一貫したワークフローを実現するための再利用可能なテンプレートを作成します：

1. 「プロジェクト設定」パネルで必要な設定をすべて行います
2. 下部の **「プロジェクトテンプレートを保存」** セクションまでスクロールします
3. わかりやすいテンプレート名を入力します（例：「Survey3N\_RGN\_Agriculture」）
4. 保存アイコンをクリックします

**メリット：**

* 複数のプロジェクトで同一の設定を適用できます
* チームメンバーと設定を共有できます
* 繰り返し行う調査において一貫性を維持できます

### 新規プロジェクトでのテンプレートの読み込み

新しいプロジェクトを作成する際：

1. メインメニューから **「新規プロジェクト」** を選択します
2. **「テンプレートから読み込む」** オプションを選択します
3. 保存済みのテンプレートを選択します
4. すべての設定が自動的に適用されます

### 作業ディレクトリ

**「プロジェクト保存フォルダ」**設定では、新しいプロジェクトがデフォルトで作成される場所を指定します：

* **デフォルトの場所**：`C:\Users\[Username]\Chloros Projects`
* **場所を変更**：編集アイコンをクリックし、新しいフォルダを選択します
* **変更するタイミング**:
  * チームでの共同作業のためのネットワークドライブ
  * より多くのストレージ容量がある別のドライブ
  * 年別/クライアント別に整理されたフォルダ構造

## PPK (Post-Processed Kinematic) の設定

正確な位置情報取得のためにGPS搭載のMAPIR DAQレコーダーを使用する場合:

### 前提条件

* GPS（GNSS）モジュール搭載のMAPIR DAQ
* 露光ピン情報が記載された.daqログファイル
* 撮影セッション中にDAQの露光ピンに接続されたカメラ

### 設定手順

1. .daqログファイルをプロジェクトフォルダに配置します
2. プロジェクト設定で、**「PPK補正を適用」**チェックボックスを有効にします
3. 必要に応じて **「光センサーのタイムゾーンオフセット」** を設定します（デフォルト：UTC の場合 0）
4. カメラを露光ピンに割り当てます：
   * **シングルカメラ**：自動的にピン 1 に割り当てられます
   * **デュアルカメラ**：各カメラを正しいピンに手動で割り当てます**露光ピンの割り当て：*** **露出ピン 1**: ドロップダウンからカメラモデルを選択
* **露出ピン 2**: 2台目のカメラを選択するか、「使用しない」を選択
* 同じカメラを両方のピンに割り当てることはできません

{% hint style="warning" %}
**重要**: 露出ピンは、それぞれのカメラに正しく割り当てる必要があります。割り当てが間違っていると、位置情報が誤ったデータになります。
{% endhint %}

***

## 高度なシナリオ

### マルチカメラプロジェクト

1つのプロジェクトで複数のMAPIRカメラからの画像を処理する場合：

1. Chlorosが各カメラモデルを自動的に検出します
2. 各カメラに適切な処理プロファイルが適用されます
3. PPK：各カメラを正しい露出ピンに手動で割り当てます
4. すべてのカメラで同じエクスポート形式とインデックスを使用します

**例**：Survey3W、RGN、および Survey3N、OCN によるデュアルカメラリグ

### タイムラプスまたは複数日間の調査

同じエリアを長期にわたり繰り返し調査する場合：

1. 標準設定を含むテンプレートを作成する
2. 各セッションで一貫したキャリブレーションターゲットの設定を使用する
3. 各日付のデータを別々のプロジェクトとして処理する
4. 比較可能な結果を得るために同一の設定を使用する
5. 経時的な分析のために同じ形式でエクスポートする

### 大規模なデータセット

画像数が多い（500枚以上）プロジェクトの場合：

* 日付やエリアごとに小規模なプロジェクトに分割することを検討してください
* 処理を高速化するには、Chloros+の並列処理を使用してください
* バッチ処理の自動化には、CLIまたはAPIの使用を検討してください
* ターゲット検出時間を短縮するために、最小再キャリブレーション間隔を調整してください

## 設定の確認

処理を開始する前に、以下の主要な設定を確認してください：

* [ ] ファイルブラウザでカメラモデルが正しく検出されている
* [ ] ヴィネット補正が有効になっている
* [ ] 反射率キャリブレーションが有効になっている
* [ ] 少なくとも1枚のキャリブレーションターゲット画像がインポートされている
* [ ] 必要なマルチスペクトルインデックスが追加されている
* [ ] ワークフローに適したエクスポート形式が選択されている
* [ ] PPK設定が構成されている（露出イベントを含む.daqファイルを使用する場合）
