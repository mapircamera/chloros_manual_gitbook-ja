# 処理の開始

画像のインポート、キャリブレーションターゲットのマーク付け、プロジェクト設定の完了後、処理を開始する準備が整います。このページでは、Chloros 処理パイプラインの起動手順について説明します。

## 処理前の確認事項

「開始」ボタンをクリックする前に、すべての準備が整っていることを確認してください：

* [ ] **ファイルのインポート完了** - すべての画像がファイルブラウザに表示されている
* [ ] **ターゲット画像のマーク付け完了** - キャリブレーション画像の「ターゲット」列にチェックが入っている
* [ ] **カメラモデルの検出完了** - 「カメラモデル」列に正しいカメラが表示されている
* [ ] **設定の構成完了** - プロジェクト設定を確認・調整済み
* [ ] **インデックスの選択** - 必要なマルチスペクトルインデックスが追加されている（必要な場合）
* [ ] **エクスポート形式の選択** - ワークフローに適した出力形式が選択されている

{% hint style="info" %}
**ヒント**: 処理を開始する前に、ファイルブラウザでいくつかの画像をクリックして、正しく読み込まれていることを確認してください。
{% endhint %}

***

## 処理の開始

### 開始ボタンの場所

「開始/再生」ボタンは、Chlorosの上部ヘッダーバーにあります：

* 位置：ウィンドウの上部中央
* アイコン：**再生/開始ボタン** <img src="../.gitbook/assets/image (2) (1).png" alt="" data-size="line">
* 状態：処理の準備が整うと、ボタンが有効（点灯）になります

### クリックして開始

1. 上部ヘッダーにある**再生/開始ボタン**をクリックします
2. 処理が直ちに開始されます
3. 処理中はボタンが無効（グレー表示）になります
4. 進行状況バーが更新され、処理状況が表示されます

{% hint style="success" %}
**処理開始**：クリックすると、Chlorosがターゲット検出、デベイヤー処理、キャリブレーション、インデックス計算、エクスポートといったすべての処理ステップを自動的に実行します。
{% endhint %}

***

## 処理モードについて

Chlorosは、ライセンスに応じて2つの異なる処理モードで動作します：

### 無料モード（順次処理）

**すべてのユーザーが利用可能**

**動作原理：**

* 画像を1枚ずつ順次処理します
* シングルスレッド動作
* メモリ使用量が少ない

**進行状況バーには2つの段階が表示されます：**

1.**ターゲット検出** - キャリブレーションターゲットのスキャン
2. **処理** - キャリブレーションの適用と画像のエクスポート**処理時間：**

* Chloros+の並列モードよりも大幅に遅い
* 小規模から中規模のデータセット（200枚未満）に適しています

### Chloros+ モード（並列処理）

**Chloros+ ライセンスが必要です**

**動作原理：**

* [4スレッド処理パイプライン](../processing-architecture/processing-pipeline.md) を使用して複数の画像を同時に処理します
* [動的演算適応](../processing-architecture/dynamic-compute-adaptation.md) により、お使いのハードウェアに最適な戦略を自動的に選択します
* NVIDIA グラフィックスカード（デスクトップおよび Jetson）による GPU (CUDA) アクセラレーション
* Jetson Nano（1 ワーカー）から、12GB 以上の GPU を搭載したデスクトップ（3～4 ワーカー）まで拡張可能です

**プログレスバーは4つの段階を表示します**（4つのパイプラインスレッドに対応）：

1. **検出**（スレッド1） - キャリブレーションターゲットの検索
2. **分析**（スレッド2） - 画像メタデータの検証とキャリブレーションの計算
3. **キャリブレーション**（スレッド3） - GPUによるデベイヤー処理、ヴィネット補正、インデックス計算
4. **エクスポート**（スレッド4） - 処理済み画像とインデックスの保存**プログレスバーの操作：*** バーに**マウスをホバー**させると、4段階の詳細ドロップダウンパネルが表示されます
* プログレスバーを**クリック**すると、ドロップダウンパネルが固定されます
* **再度クリック**すると、固定が解除されパネルが非表示になります**処理時間:**

* 無料版モードよりも大幅に高速
* CPUコア数に応じて処理速度が向上
* GPUアクセラレーションによりさらに高速化

{% hint style="info" %}
**Chloros+ 速度**：大規模なデータセットの場合、並列処理は順次モードよりも5～10倍高速になる可能性があります。無料モードで2時間かかる500枚の画像プロジェクトも、Chloros+ を使用すれば15～20分で完了する可能性があります。
{% endhint %}

## 処理中の動作

### ステージ1：ターゲット検出

**Chlorosの動作：**

* マークされたターゲット画像（マークがない場合は全画像）をスキャン
* 各ターゲット内の4つのキャリブレーションパネルを特定
* ターゲットパネルから反射率値を抽出
* キャリブレーションのスケジュール管理のためにターゲットのタイムスタンプを記録

**所要時間：** 1～30秒（ターゲットがマークされている場合）、5～30分以上（マークされていない場合）

### ステージ2：デベイヤー処理（RAW変換）

**Chlorosの機能:**

* RAWベイヤパターンデータを完全なRGB画像に変換
* 高品質なデモザイクアルゴリズムを適用
* 画像の品質とディテールを最大限に保持

**所要時間:** 画像数およびCPU速度により異なる

### ステージ3: キャリブレーション

**Chlorosの機能:*** **ヴィネット補正**: 画像の端にあるレンズの暗さを除去します
* **反射率キャリブレーション**: ターゲット反射率値を使用して正規化します
* すべてのバンド/チャンネルに補正を適用します
* タイムスタンプに基づいて、各画像に適切なキャリブレーションターゲットを使用します

**所要時間:** 処理時間の大部分を占めます

### ステージ 4: インデックス計算

**Chloros の機能:**

* 設定されたマルチスペクトルインデックス（NDVI、NDRE など）を計算します
* キャリブレーション済みの画像にバンド演算を適用します
* 選択された各インデックスに対応するインデックス画像を生成します

**所要時間:** 画像1枚あたり数秒

### ステージ5: エクスポート

**Chlorosの機能:**

* キャリブレーション済みの画像を選択した形式で保存
* 設定されたLUTカラーを使用したインデックス画像をエクスポート
* カメラモデルのサブフォルダにファイルを書き込み
* ファイル名にサフィックスを付加して元のファイル名を保持

**所要時間:** エクスポート形式およびファイルサイズにより異なります***

## 処理の動作

### 自動処理パイプライン

開始すると、パイプライン全体が自動的に実行されます:

* ユーザーの操作は不要
* 設定されたすべてのステップが順番に実行される
* 進行状況がリアルタイムで表示される

### 処理中のコンピュータ使用状況

**フリーモード:**

* CPU使用率は比較的低い（シングルスレッド）
* 他のタスクでもコンピュータの応答性は維持される
* Chlorosを最小化して、他のアプリケーションで作業しても問題ない

**Chloros+ 並列モード:**

* CPU使用率が高い（マルチスレッド、最大16コア）
* GPU アクセラレーション使用時：GPU 使用率が高い
* 処理中はコンピュータの応答が遅くなる可能性があります
* 他の CPU 負荷の高いタスクの起動は避けてください

{% hint style="warning" %}
**パフォーマンスのヒント**：最高の Chloros+ パフォーマンスを得るには、他のアプリケーションを閉じて、Chloros がシステムリソースをフルに活用できるようにしてください。
{% endhint %}

### 処理を一時停止することはできません

**重要な制限事項：**

* 一度開始した処理は一時停止できません
* 処理をキャンセルすることは可能ですが、進行状況は失われます
* 部分的な結果は保存されません
* キャンセルした場合は最初から再実行する必要があります

**計画のヒント:** 非常に大規模なプロジェクトの場合は、バッチ処理を行うか、より詳細な制御が可能な CLI の使用を検討してください。***

## 処理の監視

処理の実行中は、以下の操作が可能です:

* **進行状況バーの確認** - 全体の完了率を確認できます
* **現在のステージの確認** - 検出、分析、キャリブレーション、またはエクスポート
* **ログタブの確認** - 詳細な処理メッセージや警告を確認できます
* **完了した画像のプレビュー** - 処理中に一部のエクスポートファイルが表示される場合があります

モニタリングに関する詳細については、[処理のモニタリング](monitoring-the-processing.md)を参照してください。

## 処理の中止

処理を停止する必要がある場合：

### 中止方法

1. **停止/キャンセルボタン**（処理中は「開始」ボタンに置き換わります）を探します
2. 「停止」ボタンをクリックします
3. 処理は直ちに停止します
4. 処理済みの結果は破棄されます

### キャンセルすべき場合

**キャンセルが妥当な理由：**

* 設定が間違っていたことに気づいた場合
* 対象画像のマーク付けを忘れた場合
* 誤った画像をインポートしてしまった
* システムの動作が遅すぎる、または応答がない

**キャンセル後：**

* 問題点を確認し、修正する
* 必要に応じて設定を調整する
* 処理を最初から再開する
* 最もクリーンな状態にするには、Chlorosを完全に終了し、再起動する

{% hint style="warning" %}
**部分的な結果は保存されません**：キャンセルすると、すべての進行状況が破棄されます。Chlorosは、部分的に処理された画像を保存しません。
{% endhint %}

## 処理時間の目安

実際の処理時間は、以下の要因によって大きく異なります：

* 画像の数
* 画像の解像度
* 入力形式（RAW または JPG）
* 処理モード (無料版 vs Chloros+)
* CPUの速度とコア数
* GPUの利用可否（Chloros+のみ）
* 計算するインデックスの数
* エクスポート形式の複雑さ

### おおよその目安（Chloros+、12MP画像、最新CPU）

| 画像数 | 無料モード | Chloros+ (CPU) | Chloros+ (GPU) |
| ----------- | --------- | -------------- | -------------- |
| 50 画像   | 15～20 分 | 5～8 分        | 3～5 分        |
| 100 画像  | 30～40 分 | 10～15 分      | 5～8 分        |
| 200 画像  | 1～1.5 時間 | 20～30 分      | 10～15 分      |
| 500 画像  | 2～3 時間   | 45～60 分      | 20～30 分      |
| 1000 画像 | 4～6 時間   | 1.5～2 時間      | 40～60 分      |

{% hint style="info" %}
**初回実行**: Chlorosがキャッシュとプロファイルを構築するため、初期処理には時間がかかる場合があります。同様のデータセットのその後の処理は高速になります。
{% endhint %}

## 起動時の一般的な問題

### 開始ボタンが無効（グレー表示）

**考えられる原因:**

* 画像がインポートされていない
* バックエンドが完全に起動していない
* 前回の処理がまだ実行中である
* プロジェクトが完全に読み込まれていない

**解決策:**

1. バックエンドが完全に初期化されるまで待つ（メインメニューのアイコンを確認）
2. ファイルブラウザで画像がインポートされていることを確認する
3. ボタンが依然として無効な場合は、Chlorosを再起動する
4. デバッグログでエラーメッセージを確認する

### 処理が開始された直後に失敗する

**考えられる原因:**

* プロジェクトに有効な画像がない
* 画像ファイルが破損している
* ディスク容量が不足している
* メモリ（RAM）が不足している

**解決策:**

1. デバッグログを確認し <img src="../.gitbook/assets/icon_log.JPG" alt="" data-size="line"> にエラーメッセージがないか確認してください
2. 空きディスク容量を確認してください
3. 画像のサブセット（一部）のみを処理してみてください
4. 画像が破損していないか確認してください

### 「ターゲットが検出されませんでした」という警告

**考えられる原因:**

* ターゲット画像のマーク付けを忘れた
* ターゲット画像に視認可能なターゲットが含まれていない
* ターゲット検出の設定が厳しすぎる

**解決策:**

1. [ターゲット画像の選択](choosing-target-images.md)を確認する
2. 「ターゲット」列で適切な画像にマークを付ける
3. マークした画像内でターゲットが視認できることを確認する
4. 必要に応じてターゲット検出設定を調整する

## 処理を成功させるためのヒント

### 開始前に

1. **まずは少数の画像でテストする** - 10～20枚の画像を処理して設定を確認する
2. **空きディスク容量を確認する** - データセットサイズの2～3倍の空き容量を確保する
3. **不要なアプリケーションを閉じる** - システムリソースを解放する
4. **ターゲット画像を確認する** - マークされたターゲットをプレビューして品質を確認する
5. **プロジェクトを保存する** - プロジェクトは自動保存されますが、手動で保存することを推奨します

### 処理中

1. **システムのスリープを回避する** - 省電力モードを無効にする
2. **Chlorosを最前面に表示する** - 少なくともタスクバーに表示されるようにする
3. **進行状況を時々確認する** - 警告やエラーがないか確認する
4. **他の負荷の高いアプリケーションを起動しない** - 特に Chloros+ の並列モード使用時は注意

### Chloros+ GPU アクセラレーション

NVIDIA GPU アクセラレーションを使用する場合：

1. NVIDIA ドライバーを最新バージョンに更新する
2. GPU の VRAM が 4GB 以上であることを確認する
3. GPUを多用するアプリケーション（ゲーム、動画編集など）を閉じてください
4. GPUの温度を監視してください（十分な冷却を確保してください）
